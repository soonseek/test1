# 코드 사양 (Code Specification)

**Task ID**: task-1-2-2
**제목**: Next.js Image 컴포넌트로 이미지 최적화 구현

---

### 파일: src/components/pokemon/PokemonCard.tsx
---
설명: 포켓몬의 이미지, 번호, 이름을 표시하는 개별 카드 컴포넌트
- `next/image`의 `Image` 컴포넌트를 임포트하여 사용
- `src` prop에는 포켓몬의 공식 이미지 URL을 전달
- `width`와 `height` prop을 사용하여 이미지 크기를 명시적으로 지정 (권장: 200x200 또는 300x300)
- `placeholder="blur"` prop을 적용하여 로딩 시 흐릿한 플레이스홀더 표시
- `sizes` prop을 설정하여 반응형 이미지 최적화 지원 (예: `(max-width: 768px) 100vw, 33vw`)
- 이미지 로딩 중이거나 실패했을 때를 대비한 `fallback` UI 또는 `onError` 처리 고려
- `alt` 속성에 포켓몬 이름을 포함하여 접근성(A11y) 확보
- Tailwind CSS를 사용하여 카드의 레이아웃(그림자, 테두리, 호버 효과) 스타일링

### 파일: src/app/page.tsx
---
설명: 포켓몬 목록을 그리드 형태로 렌더링하는 메인 페이지 (수정 대상)
- `PokemonCard` 컴포넌트를 임포트
- 서버 컴포넌트(Server Component)로 동작하며 데이터 페칭 로직 유지
- `PokemonCard` 사용 시 `src` URL을 prop으로 안전하게 전달
- Tailwind CSS Grid(`grid-cols-2`, `md:grid-cols-4` 등)를 사용하여 반응형 레이아웃 구성

### 파일: next.config.mjs (또는 next.config.js)
---
설명: Next.js 이미지 최적화를 위한 외부 도메인 설정
- `images.remotePatterns` 설정에 PokeAPI의 이미지 도메인(`raw.githubusercontent.com`)을 추가하여 외부 이미지 로딩 허용
- 필요시 `images.deviceSizes` 및 `images.imageSizes`를 조정하여 최적화된 이미지 크기 생성

### 파일: src/types/pokemon.ts
---
설명: 포켓몬 데이터 구조에 대한 타입 정의
- `PokemonCard` 컴포넌트에 전달할 props의 인터페이스 정의 (`id`, `name`, `imageSrc` 등)
