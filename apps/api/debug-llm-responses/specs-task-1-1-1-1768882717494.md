# 코드 사양 (Code Specification)

**Task ID**: task-1-1-1
**주제**: PokeAPI 응답 데이터 타입 정의

---

### 파일: `src/types/pokemon.ts`
---
설명: PokeAPI v2의 응답 구조를 기반으로 포켓몬 데이터를 위한 TypeScript 인터페이스를 정의합니다. 이 타입들은 API Route 및 클라이언트 컴포넌트 전반에서 사용됩니다.

- **IPokemonListItem**: PokeAPI 리스트의 개별 항목 매핑
  - `name`: string (포켓몬 이름, 예: 'bulbasaur')
  - `url`: string (상세 정보 조회 URL)
- **IPokemonListResponse**: PokeAPI 리스트 전체 응답 매핑
  - `count`: number (전체 포켓몬 개수)
  - `next`: string | null (다음 페이지 URL)
  - `previous`: string | null (이전 페이지 URL)
  - `results`: IPokemonListItem[] (포켓몬 배열)
- **IPokemon**: (선택 사항) 향후 상세 조회를 대비한 기본 타입 힌트
  - `id`: number
  - `name`: string
  - `types`: array
  - `sprites`: object

### 파일: `src/lib/utils/pokemon.ts`
---
설명: 포켓몬 데이터 처리를 위한 유틸리티 함수를 모아두는 라이브러리 파일입니다. (향후 Story 1-2에서 구현될 로직을 대비하여 자리 표시)

- **extractPokemonId**: URL에서 ID를 추출하는 함수 정의 (예: `url`에서 ID 파싱 로직)
- 타입 안전성을 보장하기 위해 `src/types/pokemon.ts`의 인터페이스를 import하여 사용

### 파일: `src/app/api/pokemon/route.ts`
---
설명: 클라이언트의 요청을 받아 PokeAPI v2의 `pokemon` 엔드포인트(`https://pokeapi.co/api/v2/pokemon`)로 데이터를 요청하고, 결과를 반환하는 Next.js API Route 핸들러입니다.

- HTTP GET 메서드 지원
- Next.js Fetch API(`fetch`) 또는 Axios를 사용하여 외부 API 호출
- 쿼리 파라미터(`offset`, `limit`) 처리 로직 포함
- 응답 데이터 타입을 `IPokemonListResponse`로 캐스팅하여 반환
- 에러 발생 시 적절한 HTTP 에러 코드(500 등)와 메시지 반환
