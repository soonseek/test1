## 발견된 주요 패턴

### 1. Agent 워크플로우

**순차적 실행:**
```
RequirementAnalyzer → EpicStory → ScrumMaster →
Developer → CodeReviewer → Tester →
PromptBuilder → CodeGenerator →
GitHubPusher → NetlifyDeployer → E2ETestRunner
```

**이벤트 기반 트리거:**
- survey.submitted → RequirementAnalyzer
- requirement.completed → EpicStory
- epic-story.completed → ScrumMaster
- task.assigned → Developer

### 2. Monorepo 구조

**pnpm workspace:**
```yaml
packages:
  - 'apps/*'
  - 'packages/*'
```

**공유 의존성:**
- packages/db (모든 패키지)
- packages/shared (타입)
- packages/agent-framework (Agent 시스템)

### 3. API 아키텍처

**RESTful 설계:**
- /api/projects - 프로젝트 CRUD
- /api/magic/* - MVP 생성 워크플로우
- /api/survey/* - 설문조사
- /api/upload/* - 파일 업로드

### 4. 데이터 모델링

**Prisma ORM:**
- Project (프로젝트)
- SessionFile (업로드된 파일)
- SurveyAnswer (설문조사 답변)
- Deployment (배포 정보)
- IssueReport (이슈 리포트)
- AgentExecution (Agent 실행 기록)

**관계:**
- Project 1:N SessionFile
- Project 1:1 SurveyAnswer
- Project 1:1 Deployment
- Project 1:N AgentExecution
