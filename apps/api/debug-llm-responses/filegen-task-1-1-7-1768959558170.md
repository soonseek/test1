```typescript
import { PokemonListResponse } from '@/types/pokemon';

const POKEAPI_BASE_URL = 'https://pokeapi.co/api/v2/pokemon';

/**
 * PokeAPI로부터 포켓몬 리스트를 가져오는 함수
 * @param limit 가져올 아이템 개수 (기본값: 20)
 * @param offset 시작 위치 (기본값: 0)
 */
export async function fetchPokemonList(
  limit: number = 20,
  offset: number = 0
): Promise<PokemonListResponse> {
  const url = `${POKEAPI_BASE_URL}?limit=${limit}&offset=${offset}`;

  const response = await fetch(url);

  if (!response.ok) {
    throw new Error(`Failed to fetch pokemon list: ${response.statusText}`);
  }

  const data: PokemonListResponse = await response.json();
  return data;
}
```

## 파일: src/app/api/pokemon/route.ts

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { fetchPokemonList } from '@/lib/utils/api';

export async function GET(request: NextRequest) {
  try {
    const searchParams = request.nextUrl.searchParams;
    
    // 쿼리 파라미터 파싱 (기본값 설정)
    const limit = Number(searchParams.get('limit')) || 20;
    const offset = Number(searchParams.get('offset')) || 0;

    // 유틸리티 함수 호출
    const data = await fetchPokemonList(limit, offset);

    // 클라이언트에게 JSON 반환
    return NextResponse.json(data, { status: 200 });
  } catch (error) {
    console.error('API Error:', error);
    
    const errorMessage = error instanceof Error ? error.message : 'Unknown error occurred';
    
    return NextResponse.json(
      { error: 'Internal Server Error', message: errorMessage },
      { status: 500 }
    );
  }
}
```

# API 로컬 테스트 가이드

이 문서는 `/api/pokemon` 엔드포인트를 로컬 환경에서 테스트하는 방법을 설명합니다.

## 1. 서버 실행

프로젝트 루트 디렉토리에서 다음 명령어를 실행하여 개발 서버를 시작합니다.

```bash
npm run dev
```

서버가 정상적으로 시작되면 터미널에 `Ready in ...ms`와 같은 메시지가 뜹니다.

## 2. 엔드포인트 접속

브라우저나 Postman을 사용하여 다음 URL로 요청을 보냅니다.

**기본 URL:**
```
http://localhost:3000/api/pokemon
```

**쿼리 파라미터 예시 (선택 사항):**
```
http://localhost:3000/api/pokemon?limit=10&offset=0
```

## 3. 예상 응답 결과 (JSON)

요청이 성공하면(HTTP 200), 아래와 같은 형식의 JSON 데이터를 반환해야 합니다.

## 4. 성공 기준

테스트가 성공했는지 확인하기 위해 다음 사항을 체크하세요:

1. **상태 코드**: HTTP 200 OK
2. **데이터 구조**: `results` 배열 내부에 객체들이 존재해야 합니다.
3. **데이터 내용**: 각 객체는 `name` (문자열)과 `url` (문자열) 속성을 포함해야 합니다.
