# 코드 사양 (Code Specification)

**Task ID**: task-1-1-4
**제목**: Next.js 캐싱 전략 적용 (Revalidate)

---

### 파일: `src/lib/api/pokemon.ts`
---
설명: PokeAPI v2와 통신하여 포켓몬 데이터를 가져오는 핵심 로직을 담당하는 모듈입니다. Next.js의 내장 캐싱 메커니즘을 활용하여 서버 부하를 줄이고 응답 속도를 개선합니다.

- `PokemonListResponse` 인터페이스를 정의합니다. (PokeAPI의 `results` 배열 구조 매핑)
- `getPokemonList` 비동기 함수를 구현합니다.
- `fetch('https://pokeapi.co/api/v2/pokemon')` 호출 시 옵션 객체를 전달합니다.
- 옵션 객체 내부에 `next: { revalidate: 3600 }`을 설정하여 데이터를 1시간(3600초) 동안 캐싱하도록 구성합니다.
- 캐시 유효 기간이 지나면 다음 요청 시 백그라운드에서 데이터를 재검증(Revalidation)하여 최신화합니다.
- 에러 발생 시를 대비해 기본 에러 처리 로직을 포함하거나 에러를 상위로 던집니다.

### 파일: `src/types/pokemon.ts`
---
설명: 포켓몬 관련 데이터의 타입 안전성을 보장하기 위한 TypeScript 타입 정의 파일입니다.

- `PokemonListItem` 인터페이스를 정의합니다. (`name`: string, `url`: string 속성 포함)
- `PokemonListResponse` 인터페이스를 정의합니다. (`count`: number, `next`: string | null, `previous`: string | null, `results`: PokemonListItem[] 포함)
- 이 타입들은 API 레이어와 UI 컴포넌트 간의 데이터 계약을 보장합니다.

### 파일: `src/app/page.tsx`
---
설명: 포켓몬 도감의 메인 진입점으로, Server Component 특성을 활용해 초기 데이터를 효율적으로 렌더링합니다.

- `getPokemonList` 함수를 호출하여 포켓몬 목록 데이터를 가져옵니다.
- 가져온 데이터를 기반으로 사용자에게 포켓몬 카드 리스트를 렌더링합니다.
- 데이터 로딩 중이거나 실패했을 때의 상태를 처리하기 위한 기본 UI 구조(Skeleton 등)를 고려합니다.
- (참고) 이 파일은 데이터를 소비하는 역할을 하며, 실제 캐싱 로직은 `src/lib/api/pokemon.ts`에 위임됩니다.
