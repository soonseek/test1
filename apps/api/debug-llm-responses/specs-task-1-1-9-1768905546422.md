# 코드 사양 (Code Specification)

**Task ID**: task-1-1-9
**제목**: API Route 로컬 테스트
**목표**: `/api/pokemon` 엔드포인트를 구현하고, 쿼리 파라미터(`offset`, `limit`)를 처리하여 PokeAPI 데이터를 반환하는 로직을 사양화합니다.

---

### 파일: src/types/pokemon.ts
---
설명: PokeAPI의 응답 데이터와 우리 애플리케이션에서 사용할 포켓몬 데이터의 타입을 정의합니다.
- `PokemonListResponse` 인터페이스 정의 (count, next, previous, results)
- `PokemonListItem` 인터페이스 정의 (name, url)
- 유틸리티 타입 또는 ID 추출 로직 타입 정의 (옵션)

### 파일: src/lib/api/pokeapi.ts
---
설명: 외부 PokeAPI(v2)와 통신하는 클라이언트 로직을 담당합니다.
- `fetchPokemonList(offset: number, limit: number)` 함수 구현
- Next.js 내장 `fetch` API 사용 (캐싱 전략 고려)
- 받아온 JSON 데이터를 `PokemonListResponse` 타입으로 반환
- 네트워크 에러 처리 및 예외 던지기

### 파일: src/app/api/pokemon/route.ts
---
설명: 클라이언트의 요청을 받아 포켓몬 리스트 데이터를 반환하는 API Route 핸들러입니다.
- HTTP GET 메서드 핸들러 구현 (`export async function GET`)
- URL 쿼리 파라미터(`offset`, `limit`) 파싱 및 기본값 설정 (offset=0, limit=20)
- 파싱된 파라미터를 사용하여 `fetchPokemonList` 함수 호출
- 응답 데이터를 `Response.json()` 형태로 반환
- 에러 발생 시 적절한 HTTP 상태 코드(500 등)와 에러 메시지 반환

### 파일: src/app/api/pokemon/route.test.ts (선택 사항)
---
설명: API Route의 동작을 검증하기 위한 테스트 파일입니다 (로컬 테스트 단계에서는 Postman/브라우저를 주로 사용하지만, 자동화를 위한 준비).
- `GET` 요청 시 정상적인 JSON 응답을 반환하는지 확인
- 쿼리 파라미터에 따라 결과가 달라지는지 확인
- 타입 검증 로직 포함 가능
