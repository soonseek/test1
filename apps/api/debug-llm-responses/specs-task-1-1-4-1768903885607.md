# 코드 사양 (Code Specification)

**Task ID**: task-1-1-4
**제목**: 데이터 매핑 및 ID 추출 로직 추가

---

### 파일: `src/lib/utils/pokemon.ts`
---
설명: PokeAPI 데이터 처리를 위한 유틸리티 함수 모음
- **`getPokemonId(url: string): number`**:
  - PokeAPI의 URL(예: `https://pokeapi.co/api/v2/pokemon/1/`)을 파싱하여 숫자형 ID(예: `1`)를 반환합니다.
  - URL 끝의 슬래시(`/`)를 제거하고 마지막 경로 segment를 추출하는 로직을 포함합니다.
- **`mapPokemonList(apiResponse: PokeAPIListResponse): PokemonListItem[]`**:
  - API로부터 받은 원본 리스트 데이터를 클라이언트에 필요한 형태(`{ id, name }`)로 변환합니다.
  - 내부적으로 `getPokemonId`를 사용하여 ID를 생성합니다.

### 파일: `src/types/pokemon.ts`
---
설명: 포켓몬 관련 데이터의 TypeScript 인터페이스 정의
- **`PokeAPIListResponse`**:
  - PokeAPI v2의 `/pokemon?limit=...&offset=...` 엔드포인트 응답 형태입니다.
  - `count: number`, `next: string | null`, `previous: string | null`, `results: Array<{ name: string, url: string }>` 속성을 포함합니다.
- **`PokemonListItem`**:
  - 애플리케이션에서 사용하는 표준화된 포켓몬 아이템 타입입니다.
  - `id: number` (추출된 ID), `name: string` 속성을 포함합니다.

### 파일: `src/lib/api/pokemon.ts`
---
설명: PokeAPI 호출 및 데이터 변환을 담당하는 API 레이어
- **`fetchPokemonList(limit: number, offset: number)`**:
  - Next.js Fetch API를 사용하여 PokeAPI(`https://pokeapi.co/api/v2/pokemon`)에 GET 요청을 보냅니다.
  - 응답받은 JSON 데이터를 `mapPokemonList` 유틸리티를 통해 `PokemonListItem[]` 형태로 변환하여 반환합니다.
  - 추후 캐싱 로직(Prisma 등)이 추가될 수 있도록 함수 구조를 유지합니다.
