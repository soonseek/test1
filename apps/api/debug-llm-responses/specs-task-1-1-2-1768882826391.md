# 코드 사양 (Code Specification)

**Task ID**: task-1-1-2
**대상**: API 라우트 핸들러 생성 및 타입 정의

---

### 파일: src/types/pokemon.ts
---
설명: PokeAPI의 응답 데이터와 우리 애플리케이션에서 사용할 데이터 형태를 정의하는 TypeScript 인터페이스 모음입니다.
- `PokeApiListResponse`: PokeAPI v2 `pokemon?limit=...` 엔드포인트의 원본 응답 형태 정의 (`count`, `next`, `previous`, `results` 포함)
- `PokeApiListItem`: 리스트 내 개별 아이템 형태 (`name`, `url` 포함)
- `PokemonListItem`: 클라이언트로 반환할 최종 데이터 형태 정의 (ID 추출 및 필요한 필드만 포함)
- 유틸리티 타입 또는 헬퍼 함수가 필요할 경우 URL에서 ID를 추출하는 로직을 위한 타입 정의 포함

### 파일: src/lib/api/pokemon.ts
---
설명: PokeAPI와의 통신을 담당하는 서버 사이드 유틸리티 함수입니다. (선택 사항이나 권장됨)
- `fetchPokemonList(limit: number, offset: number)` 함수 구현
- Next.js의 `fetch` API를 사용하여 PokeAPI(`https://pokeapi.co/api/v2/pokemon`)에 요청
- 받아온 데이터를 `PokemonListItem` 형태로 매핑하여 반환
- 향후 캐싱 로직(Prisma 등)이 추가될 때를 대비해 계층 분리 유지

### 파일: src/app/api/pokemon/route.ts
---
설명: 클라이언트의 요청을 받아 포켓몬 리스트 데이터를 반환하는 API Route 핸들러입니다.
- HTTP GET 메서드 핸들러 함수 `GET` 구현
- 쿼리 파라미터(`limit`, `offset`)를 파싱 및 기본값 설정 (예: limit=20, offset=0)
- `src/lib/api/pokemon.ts`의 `fetchPokemonList` 함수 호출 또는 직접 `fetch` 로직 수행
- 응답 데이터를 `JSON` 형태로 반환
- 에러 발생 시 적절한 HTTP 상태 코드(500 등)와 에러 메시지 반환
- CORS 및 헤더 설정이 필요할 경우 처리 (Next.js 기본값 사용 가능)
