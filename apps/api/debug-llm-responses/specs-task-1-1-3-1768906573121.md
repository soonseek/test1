# 코드 사양 (Code Specification)

**Task ID**: task-1-1-3
**제목**: PokeAPI 리스트 페칭 로직 구현

---

### 파일: src/types/pokemon.ts
---
설명: PokeAPI의 응답 데이터 형태를 매핑하기 위한 TypeScript 인터페이스 정의
- `PokemonListResponse`: PokeAPI v2 `/pokemon` 엔드포인트의 루트 응답 구조 정의 (`count`, `next`, `previous`, `results`)
- `PokemonListItem`: 리스트 내 개별 포켓몬 아이템 구조 정의 (`name`, `url`)
- `NamedAPIResource`: 공통 리소스 타입 (이름, URL) 정의

### 파일: src/lib/api/pokeapi.ts
---
설명: 외부 PokeAPI와의 통신을 담당하는 비동기 함수 모듈
- `getPokemonsFromApi(offset: number, limit: number)` 함수 구현
- 내부적으로 `fetch` API 사용하여 `https://pokeapi.co/api/v2/pokemon` 요청
- Query 파라미터(`offset`, `limit`)를 URL에 동적 연결하여 전송
- 응답 데이터를 `PokemonListResponse` 타입으로 변환하여 반환
- 네트워크 에러 발생 시 에러를 던지거나 적절한 에러 처리 로직 포함

### 파일: src/app/api/pokemon/route.ts
---
설명: 클라이언트의 요청을 받아 서버 사이드에서 포켓몬 리스트를 제공하는 Next.js API Route Handler
- HTTP GET 메서드 핸들러 구현 (`export async function GET`)
- URL Search Params로부터 `offset`과 `limit` 값을 파싱 (기본값 설정 포함)
- `getPokemonsFromApi` 함수 호출을 통해 데이터 획득
- 획득한 데이터를 JSON 형식(`NextResponse.json`)으로 클라이언트에 반환
- CORS 및 에러 핸들링 기본 구조 포함
