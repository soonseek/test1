# 코드 사양 (Code Specification)

**Task ID**: task-1-1-8
**주제**: PokeAPI 데이터 캐싱 설정 (Next.js Fetch ISR)

Next.js 14+ App Router 환경에서 `fetch` 함수의 네이티브 캐싱 기능을 활용하여 ISR(Incremental Static Regeneration) 전략을 적용합니다. 이를 통해 PokeAPI 호출 빈도를 줄이고 페이지 로딩 성능을 최적화합니다.

---

### 파일: `src/lib/api/pokemon.ts`
---
설명: PokeAPI와 통신하여 데이터를 가져오는 비즈니스 로직을 담당하는 서버 전용 유틸리티 모듈입니다.
- `fetch` 함수를 사용하여 `https://pokeapi.co/api/v2/pokemon` 엔드포인트를 호출합니다.
- `fetch`의 두 번째 인자로 `{ next: { revalidate: 3600 } }` 옵션을 전달하여 데이터를 1시간(3600초) 동안 캐싱하도록 설정합니다.
- 포켓몬 리스트 데이터를 가져오는 `getPokemonList` 함수를 export 합니다.
- 필요 시 TypeScript 인터페이스(`PokemonListResponse`, `Pokemon`)를 정의하여 응답 데이터의 타입 안전성을 보장합니다.

### 파일: `src/app/page.tsx`
---
설명: 포켓몬 목록을 사용자에게 보여주는 메인 페이지 컴포넌트입니다.
- React Server Component로 동작하도록 구현합니다 (async/await 사용).
- 컴포넌트 내부 또는 별도 함수에서 `src/lib/api/pokemon.ts`의 `getPokemonList`를 호출하여 데이터를 가져옵니다.
- 가져온 데이터를 기반으로 `shadcn/ui`의 Card 컴포넌트 등을 활용하여 UI를 렌더링합니다.
- ISR이 적용되어 있으므로, 사용자 요청 시 1시간 이내의 캐시된 데이터를 즉시 반환하거나, 캐시 만료 시 백그라운드에서 재검증을 수행합니다.
