# 코드 사양 (Code Specification)

**Task ID**: task-1-2-2
**제목**: Next.js Image 컴포넌트로 이미지 최적화 구현

---

### 파일: src/components/pokemon/PokemonCard.tsx
---
설명: 포켓몬의 정보(이미지, 번호, 이름)를 시각적으로 표시하는 카드 컴포넌트입니다. Next.js의 최적화 기능을 적용합니다.
- `next/image`의 `Image` 컴포넌트를 임포트하여 사용합니다.
- `src` 속성에 PokeAPI의 공식 아트워크 URL(`sprites.other['official-artwork'].front_default`)을 연결합니다.
- 이미지 컨테이너는 `relative w-full h-full` 또는 `aspect-square`로 설정하여 비율을 유지합니다.
- `Image` 컴포넌트에 `fill` 속성을 적용하여 컨테이너를 꽉 채우도록 설정합니다.
- `sizes` 속�性을 추가하여 반응형 이미지 최적화를 지원합니다 (예: `"(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"`).
- `loading="lazy"` 속성을 추가하여 뷰포트 외 이미지의 로딩을 지연시켜 성능을 향상시킵니다.
- 접근성을 위해 `alt` 속성에는 포켓몬의 이름을 동적으로 할당합니다.
- 이미지 로딩 중이거나 실패 시의 시각적 피드백(placeholder)을 고려한 구조를 가집니다.

### 파일: src/app/layout.tsx
---
설명: 애플리케이션의 루트 레이아웃으로, 외부 이미지 도메인에 대한 Next.js의 최적화 설정을 담당합니다.
- `next.config.mjs`에 `remotePatterns`를 설정하여 `raw.githubusercontent.com` (PokeAPI 이미지 소스) 도메인을 허용합니다.
- (참고) 이 파일이 직접 수정되지 않더라도, 이미지가 정상적으로 작동하려면 프로젝트 루트의 설정 파일이 해당 도메인을 허용해야 합니다.

### 파일: next.config.mjs
---
설명: Next.js의 이미지 최적화를 위한 설정 파일입니다.
- `images.remotePatterns` 배열을 설정합니다.
- `protocol: 'https'`, `hostname: 'raw.githubusercontent.com'`을 추가하여 PokeAPI의 이미지 리소스 로딩을 허용합니다.
