# 코드 사양 (Code Specification)

**Task ID**: task-1-2-2
**제목**: PokemonCard 이미지 영역 구현 (Next.js Image)

이 문서는 `PokemonCard` 컴포넌트 내부의 이미지 영역을 구현하기 위한 파일 구조와 상세 기능 명세서입니다. Next.js 14+의 `Image` 컴포넌트 최적화 기능을 활용하여 성능과 사용자 경험을 확보하는 것을 목표로 합니다.

---

### 파일: src/components/pokemon/PokemonCard.tsx
---
설명: 포켓몬의 정보(이미지, 번호, 이름)를 시각적으로 표현하는 컴포넌트입니다. 이번 Task에서는 이미지 영역 구현에 집중합니다.
- `src/components/ui/card` (shadcn/ui)를 기반으로 하여 카드 스타일링을 적용합니다.
- `props`로 `imageUrl`, `name`, `id`를 받습니다.
- **이미지 영역 구현**:
  - `next/image`의 `Image` 컴포넌트를 임포트하여 사용합니다.
  - `src` 속성에 props로 전달받은 `imageUrl`을 적용합니다.
  - `width`와 `height`는 포켓몬 도감의 일관성을 위해 고정값(예: 200x200) 또는 비율에 맞춰 설정합니다.
  - `alt` 속성에 포켓몬 이름(`name`)을 입력하여 접근성(Accessibility)을 준수합니다.
  - 이미지 로딩 중 깜빡임(CLs)을 방지하기 위해 `placeholder="blur"`를 사용하거나, 배경색을 설정합니다.
  - 이미지가 카드 상단 중앙에 위치하도록 Flexbox 또는 Grid 레이아웃을 배치합니다.

### 파일: src/types/pokemon.ts
---
설명: 포켓몬 데이터의 인터페이스를 정의하는 타입 정의 파일입니다. 컴포넌트 간 데이터 전달의 안전성을 보장합니다.
- `PokemonListItem` 인터페이스를 정의합니다.
- 필수 프로퍼티:
  - `id`: number (포켓몬 번호)
  - `name`: string (포켓몬 이름)
  - `imageUrl`: string (이미지 URL)
- 이 타입은 `PokemonCard` 컴포넌트의 Props 타입으로 재사용됩니다.

### 파일: src/app/page.tsx
---
설명: 메인 페이지로, 구현된 `PokemonCard` 컴포넌트를 렌더링하여 동작을 테스트하고 확인하는 역할을 합니다.
- `PokemonCard` 컴포넌트를 import 합니다.
- 테스트를 위해 Mock 데이터(더미 데이터) 배열을 생성합니다. (예: 이상해씨, 파이리, 꼬부기 데이터)
- `map` 함수를 사용하여 `PokemonCard` 리스트를 그리드 형태로 렌더링합니다.
- Tailwind CSS의 `grid` 클래스를 사용하여 반응형 레이아웃(모바일 1열, 데스크톱 3~4열)을 적용합니다.

### 파일: next.config.mjs (수정 및 확인)
---
설명: Next.js 설정 파일입니다. 외부 이미지 소스(PokeAPI 등)를 최적화하여 사용하기 위해 도메인 설정이 필요합니다.
- `images.remotePatterns` 설정을 확인하거나 추가합니다.
- PokeAPI의 이미지 호스트 도메인(`raw.githubusercontent.com` 등)을 허용 목록에 추가하여 `next/image`가 외부 이미지를 로드할 수 있도록 합니다.
