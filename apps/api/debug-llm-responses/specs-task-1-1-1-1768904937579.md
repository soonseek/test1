# 코드 사양 (Code Specification)

**Task ID**: task-1-1-1
**제목**: PokeAPI 응답 데이터 타입 정의

---

### 파일: `src/types/pokemon.ts`
---
설명: PokeAPI v2의 원본 응답 형식과 애플리케이션에서 실제로 사용할 도메인 모델(Pokemon)의 TypeScript 타입을 정의하는 파일입니다. 데이터 페칭 로직과 UI 컴포넌트 간의 타입 계약을 보장합니다.

- **PokeAPIListResponse 인터페이스 정의**
  - PokeAPI `https://pokeapi.co/api/v2/pokemon?limit=...` 엔드포인트의 응답 형식 매핑
  - `count` (number): 전체 아이템 개수
  - `next` (string | null): 다음 페이지 URL
  - `previous` (string | null): 이전 페이지 URL
  - `results` (Array<{ name: string, url: string }>): 포켓몬 이름과 상세 URL을 포함한 배열

- **PokeAPIResource 인터페이스 정의**
  - API 리스트 응답 내부의 개별 항목 타입
  - `name` (string): 포켓몬 영문 이름 (예: 'bulbasaur')
  - `url` (string): 상세 정보 조회 URL (예: 'https://pokeapi.co/api/v2/pokemon/1/')

- **PokemonListItem 인터페이스 정의**
  - 앱 UI(리스트 뷰)에서 사용하는 최종 데이터 모델
  - `id` (number): 포켓몬 고유 ID (URL 파싱을 통해 추출)
  - `name` (string): 포켓몬 영문 이름
  - `imageUrl` (string): 포켓몬 이미지 URL (PokeAPI의 공식 아트워크 URL 포맷 사용: `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/other/official-artwork/{id}.png`)

- **타입 가드 및 유틸리티 타입 (선택 사항)**
  - 필요시 ID 추출 로직을 위한 헬퍼 타입 정의
