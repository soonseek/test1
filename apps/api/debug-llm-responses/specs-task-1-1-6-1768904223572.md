# 코드 사양 (Code Specification)

**Task ID**: task-1-1-6
**제목**: 클라이언트 데이터 페칭 훅 작성
**작성자**: Next.js 14+ Full-Stack Architect

---

## 개요
이 사양은 Next.js 14 App Router 환경에서 클라이언트 사이드 데이터 페칭을 담당하는 로직을 정의합니다. `SWR` 라이브러리를 사용하여 서버 상태를 관리하고, 앞서 구현된 내부 API Route(`/api/pokemons`)와의 통신을 처리합니다.

---

### 파일: src/hooks/usePokemonList.ts
---
설명: 포켓몬 목록 데이터를 가져오는 커스텀 React 훅 (SWR 기반)
- `swr` 라이브러리의 `useSWR` 훅을 사용하여 데이터 페칭 및 캐싱 로직 구현
- 내부 API 엔드포인트(`/api/pokemons`)를 호출하는 전용 fetcher 함수 정의
- API로부터 받은 데이터(JSON)를 타입 단언(Type Assertion)하여 반환
- `data`, `error`, `isLoading` 상태를 반환하여 컴포넌트에서 UI 상태 제어 가능하도록 함
- 옵션값: `revalidateOnFocus: true` (사용자가 탭을 다시 포커스할 때 데이터 갱신)

### 파일: src/types/pokemon.ts
---
설명: 포켓몬 데이터의 형태를 정의하는 TypeScript 인터페이스
- API 응답 본문(Response Body)의 구조를 정의하는 `PokemonListResponse` 인터페이스
- 개별 포켓몬 아이템의 데이터 구조를 정의하는 `PokemonListItem` 인터페이스
  - `id`: number (포켓몬 고유 번호)
  - `name`: string (포켓몬 이름)
  - `url`: string (상세 정보 조회 URL 또는 이미지 URL)
  - `types`: string[] (포켓몬 타입 배열, 예: ["fire", "flying"])
- `usePokemonList.ts` 훅에서 반환되는 데이터의 타입으로 사용하여 타입 안전성 확보

### 파일: src/lib/fetcher.ts
---
설명: SWR에서 사용할 범용적인 HTTP 요청 유틸리티 함수
- `fetch` API를 래핑한 간단한 비동기 함수
- 입력받은 URL(`string`)로 GET 요청을 보냄
- 응답이 정상적(200 OK)이지 않을 경우 에러를 throw하여 SWR의 에러 핸들링 로직 연동
- 응답 본문을 JSON 파싱하여 반환
- 향후 헤더 설정(인증 등)이 필요할 경우 이 파일에서 일관되게 처리 가능하도록 분리
