# 코드 사양 (Code Specification)

**Task ID**: task-1-2-3
**작성자**: Next.js 14+ Full-Stack Architect

---

### 파일: `src/app/page.tsx`
---
설명: 포켓몬 도감의 메인 페이지로, 서버 컴포넌트(Server Component)로 동작하며 데이터를 페칭하고 리스트 레이아웃을 구성합니다.
- `getPokemonList` 유틸리티(또는 API)를 호출하여 포켓몬 목록 데이터를 가져옵니다.
- `PokemonGrid` 컴포넌트에 데이터를 props로 전달하여 렌더링합니다.
- 페이지 메타데이터(Metadata)를 설정하여 SEO를 최적화합니다.

### 파일: `src/components/pokemon-grid.tsx`
---
설명: 포켓몬 카드 리스트를 감싸는 반응형 그리드 레이아웃 컨테이너입니다. Tailwind CSS Grid를 사용합니다.
- Tailwind CSS 클래스 `grid grid-cols-2 md:grid-cols-3 lg:grid-cols-4 gap-4`를 적용하여 반응형 레이아웃을 구현합니다.
- `gap-4` 또는 `gap-6`을 사용하여 카드 간의 간격을 조정합니다.
- 자식 컴포넌트로 `PokemonCard`를 반복 렌더링합니다.
- 데이터가 없을 경우 "데이터 없음" 메시지를 표시하는 처리를 포함합니다.

### 파일: `src/components/pokemon-card.tsx`
---
설명: 개별 포켓몬 정보를 표시하는 카드 컴포넌트입니다. 클라이언트 컴포넌트로 구현하여 인터랙션(호버 효과 등)을 다룹니다.
- shadcn/ui의 `Card` 컴포넌트 구조(Card, CardContent)를 활용합니다.
- `next/image`를 사용하여 포켓몬 이미지를 최적화하여 로드합니다 (width, height, placeholder 설정).
- 포켓몬 번호(ID), 이름을 텍스트로 표시합니다.
- 카드 클릭 시 해당 포켓몬의 상세 페이지(`/pokemon/[id]`)로 이동하는 `Link` 컴포넌트를 감싸거나 onClick 이벤트를 처리합니다.
- Tailwind 클래스를 사용하여 호버 시 그림자 효과나 스케일 변환 등의 UI 피드백을 제공합니다.

### 파일: `src/lib/utils/pokemon.ts`
---
설명: 포켓몬 데이터를 가져오는 로직을 담당하는 서버 사이드 유틸리티 함수입니다.
- PokeAPI 또는 내부 DB(Prisma)에서 데이터를 조회하는 로직을 포함합니다 (현재 태스크에서는 목업 데이터 또는 간단한 fetch 로직 사용 가능).
- `getPokemonList()` 함수를 export 하여 페이지에서 사용할 수 있게 합니다.
- 필요시 이미지 URL을 포맷팅하는 헬퍼 함수를 포함합니다.
