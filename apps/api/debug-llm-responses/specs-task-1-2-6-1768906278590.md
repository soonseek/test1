# 코드 사양 (Code Specification)

**Task ID**: task-1-2-6
**제목**: 반응형 디자인 미디어 쿼리 검증 및 수정

## 개요
이 Task는 `STORY-1-2`의 인수 조건 중 "반응형 디자인"과 "그리드 레이아웃"을 충족하기 위해 기존 또는 예정된 컴포넌트의 스타일을 명확히 정의합니다. 특히 Tailwind CSS의 유틸리티 클래스를 사용하여 모바일(sm), 태블릿(md), 데스크톱(lg) 환경에서 카드의 비율(aspect-ratio)이 깨지지 않도록 하는 데 중점을 둡니다.

---

### 파일: src/components/pokemon-card.tsx
---
설명: 포켓몬 도감의 개별 카드를 표시하는 컴포넌트
- `shadcn/ui`의 `Card` 컴포넌트를 기반으로 구성
- **비율 유지**: 이미지 컨테이너에 `aspect-square` 클래스를 적용하여 모든 디바이스에서 정사각형 비율을 강제
- **이미지 최적화**: `next/image`의 `Image` 컴포넌트를 사용하며, `fill` 속성과 `object-fit` 설정을 통해 부모 컨테이너에 꽉 차게 렌더링
- **반응형 내부 요소**: 카드 내부의 텍스트(번호, 이름)가 모바일에서도 잘리지 않도록 `truncate` 처리 및 적절한 `padding` 적용
- **링크 연결**: 카드 전체 또는 특정 영역 클릭 시 상세 페이지(`/pokemon/[id]`)로 이동하는 `Link` 컴포넌트로 감싸거나 `onClick` 이벤트 처리

### 파일: src/app/page.tsx
---
설명: 포켓몬 목록을 그리드 형태로 보여주는 메인 페이지 (Server Component)
- **반응형 그리드 레이아웃**:
  - `grid-cols-2`: 모바일(Small, 640px 이하) 화면에서 2열 배치
  - `md:grid-cols-3`: 태블릿(Medium, 768px 이상) 화면에서 3열 배치
  - `lg:grid-cols-4`: 데스크톱(Large, 1024px 이상) 화면에서 4열 배치
- **간격(Gap) 관리**: `gap-4` 또는 `gap-6`을 사용하여 디바이스 크기에 따라 적절한 여백 유지
- **데이터 로딩**: Server Component 환경에서 데이터를 가져와 `PokemonCard` 컴포넌트 리스트를 렌더링
- **컨테이너 제한**: `max-w-screen-2xl`과 `mx-auto`를 사용하여 초대형 화면에서도 레이아웃이 중앙에 위치하도록 제어

### 파일: src/components/ui/card.tsx (shadcn/ui)
---
설명: 카드 UI의 기본 스타일을 정의하는 컴포넌트 (존재 시 확인 및 수정)
- `rounded-xl`, `border`, `bg-card`, `text-card-foreground`, `shadow-sm` 등의 기본 클래스 포함
- `className` prop을 통해 외부에서 추가적인 스타일(예: `hover:scale-105` 트랜지션)을 주입할 수 있도록 구현
- (참고: 이 파일은 shadcn CLI로 설치되었다고 가정하며, Task 범위 내에서 직접 수정보다는 사용을 전제로 함)

### 파일: tailwind.config.ts
---
설명: 프로젝트 전체의 Tailwind CSS 설정을 관리
- `content` 배열에 `./src/**/*.{ts,tsx}`가 포함되어 있어 `pokemon-card.tsx` 및 `page.tsx`의 클래스가 정상적으로 빌드되는지 확인
- `theme.extend`에 프로젝트 특화 색상(포켓몬 타입별 색상 등)이 필요할 경우 확장 가능성을 고려하나, 본 Task에서는 기본 설정 사용
