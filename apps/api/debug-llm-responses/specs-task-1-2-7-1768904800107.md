# 코드 사양 (Code Specification)

## 개요
이 사양은 `STORY-1-2`의 인수 조건(AC3: 반응형 디자인)을 충족하기 위해, 포켓몬 카드 리스트가 모바일(320px+), 태블릿(768px+), 데스크톱(1024px+) 환경에서 그리드가 깨지지 않고 최적의 카드 크기를 유지하도록 기존 컴포넌트와 스타일을 수정하는 것을 목적으로 합니다.

---

## 파일 사양

### 파일: `src/components/pokemon-card.tsx`
---
설명: 개별 포켓몬 카드의 시각적 요소를 정의하며, 반응형 환경에서 카드 내부 콘텐츠가 깨지지 않도록 보장합니다.
- **이미지 최적화**: `next/image`의 `fill` 또는 `responsive` 속성을 사용하여 부모 컨테이너 크기에 맞춰 이미지가 자동 조절되도록 설정 (320px 모바일 환경 대응).
- **텍스트 오버플로우 방지**: 포켓몬 이름이 길 경우 말줄임표(...) 처리하도록 Tailwind 클래스(`truncate`, `text-ellipsis`) 적용.
- **카드 비율 유지**: `aspect-ratio` 유틸리티를 사용하여 이미지 영역의 높이가 너비에 따라 일정하게 유지되도록 수정.
- **호버 효과**: 데스크톱 환경에서만 적용되는 호버(확대) 효과가 모바일 레이아웃에 영향을 주지 않도록 조정.

### 파일: `src/components/pokemon-grid.tsx`
---
설명: 카드 리스트를 배치하는 그리드 레이아웃을 담당하며, 뷰포트 크기에 따라 열(Column) 개수를 동적으로 조정합니다.
- **모바일 (320px+)**: 화면 폭이 좁을 경우 1열(`grid-cols-1`) 또는 2열(`grid-cols-2`)로 배치하여 카드가 너무 작아지지 않도록 설정.
- **태블릿 (768px+)**: `md:` 브레이크포인트를 사용하여 3열(`md:grid-cols-3`) 또는 4열(`md:grid-cols-4`)로 배치.
- **데스크톱 (1024px+)**: `lg:` 브레이크포인트를 사용하여 5열 이상(`lg:grid-cols-5`) 배치하여 공간 효율성 극대화.
- **간격(Gap) 조정**: `gap-4` 또는 `gap-6`을 사용하여 반응형 크기에 따라 카드 간격이 자동으로 조정되도록 설정.

### 파일: `src/app/page.tsx`
---
설명: 메인 페이지의 레이아웃 컨테이너 역할을 하며, 전체 페이지의 최대 너비와 패딩을 제어합니다.
- **컨테이너 제한**: `max-w-screen-2xl` 또는 `max-w-7xl` 클래스를 적용하여 초와이드 화면에서도 카드가 무한정 퍼지지 않도록 제한.
- **패딩(Padding) 설정**: 모바일에서는 `px-4`, 데스크톱에서는 `px-8` 등을 적용하여 화면 가장자리 여백 확보.
- **Grid 컴포넌트 렌더링**: `PokemonGrid` 컴포넌트를 렌더링하고, 필요한 데이터를 props로 전달하는 로직 유지.

### 파일: `tailwind.config.ts`
---
설명: 프로젝트 전체의 반응형 디자인 기준을 정의하는 설정 파일입니다.
- **브레이크포인트 확인**: 기본 제공되는 `screens` 설정(`sm: 640px`, `md: 768px`, `lg: 1024px`)이 Task 요건(320px+, 768px+, 1024px+)과 부합하는지 확인 및 수정.
- (선택사항) 커스텀 간격이나 색상이 필요할 경우 `theme` 확장.

## 검증 가이드 (Verification Guide)
1. **모바일 검증**: 브라우저 개발자 도구를 사용하여 320px ~ 480px 너비에서 카드가 1~2열로 정렬되고 내용이 잘리지 않는지 확인.
2. **태블릿 검증**: 768px 너비에서 그리드가 3~4열로 부드럽게 전환되는지 확인.
3. **데스크톱 검증**: 1024px 이상에서 카드가 적절한 개수(5열 이상)로 배치되고 중앙 정렬되는지 확인.
