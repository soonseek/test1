### 파일: src/types/pokemon.ts
---
설명: PokeAPI로부터 받아오는 데이터의 형태를 정의하는 타입 스크립트 인터페이스 모듈입니다. 데이터의 무결성을 보장하고 IDE의 자동 완성을 지원합니다.
- **NamedResource 인터페이스**: 포켓몬의 기본 정보(이름, URL)를 정의합니다.
- **PokemonListResponse 인터페이스**: API 응답 전체 구조(count, next, previous, results)를 정의합니다.
- **FetchPokemonListParams 인터페이스**: `fetchPokemonList` 함수의 매개변수 타입(offset, limit)을 정의합니다.

### 파일: src/lib/api/pokemon.ts
---
설명: PokeAPI v2 엔드포인트와 통신하여 포켓몬 목록 데이터를 가져오는 비즈니스 로직을 담당하는 유틸리티 함수입니다.
- **fetchPokemonList 함수 구현**:
  - `offset`과 `limit`을 매개변수로 받습니다.
  - PokeAPI(`https://pokeapi.co/api/v2/pokemon`)에 GET 요청을 보냅니다.
  - **캐싱 전략 적용**: Next.js의 `fetch` 옵션으로 `next: { revalidate: 3600 }`을 설정하여 1시간 동안 데이터 캐싱을 수행합니다. (Story 1-1 인수 조건 충족)
  - 네트워크 에러 발생 시 에러를 던지도록 처리합니다.
- **타입 안전성 보장**: `src/types/pokemon.ts`에서 정의한 타입을 사용하여 반환 및 요청 데이터를 타이핑합니다.

### 파일: src/lib/constants.ts
---
설명: 애플리케이션 전체에서 사용하는 상수 값을 관리하는 파일입니다. API 요청 중복을 방지하고 매직 넘버를 제거합니다.
- **POKEAPI_BASE_URL**: PokeAPI의 기본 URL (`https://pokeapi.co/api/v2`)을 상수로 export 합니다.
- **DEFAULT_FETCH_LIMIT**: 목록 조회 시 기본으로 가져올 아이템 개수(예: 20개)를 정의합니다.

### 파일: src/lib/utils/error.ts
---
설명: API 호출 등 비동기 작업에서 발생할 수 있는 에러를 구조화하여 관리하는 유틸리티입니다.
- **ApiError 클래스 (선택 사항)**: 표준 Error 객체를 확장하여 상태 코드나 에러 메시지를 포함한 커스텀 에러 클래스를 정의합니다. (향후 에러 바운더리 구현 시 대비)
