# 코드 사양 (Code Specification)

**Task ID**: task-1-1-3
**제목**: 포켓몬 리스트 페칭 Server Action 함수 구현 (Fetch)

---

### 파일: `src/types/pokemon.ts`
---
설명: PokeAPI v2 응답 구조에 맞춘 TypeScript 인터페이스 정의
- `PokemonListItem`: 개별 포켓몬의 이름과 URL을 포함하는 인터페이스 (`name`, `url`)
- `PokemonListResponse`: PokeAPI 리스트 응답 전체를 감싸는 인터페이스 (`count`, `next`, `previous`, `results`)
- `getPokemonList` 함수의 반환 타입으로 사용되어 타입 안전성 보장

### 파일: `src/app/actions/fetchPokemon.ts`
---
설명: PokeAPI에서 포켓몬 목록을 가져오는 Next.js Server Action 구현
- `'use server'` 지시자를 사용하여 서버 전용 함수임을 선언
- 비동기 함수 `getPokemonList(offset: number = 0, limit: number = 20)` 정의
- 내부적으로 `fetch('https://pokeapi.co/api/v2/pokemon')` 호출
- Query Parameter로 `offset`과 `limit` 전달
- Next.js의 `fetch` 캐싱 전략 적용: `next: { revalidate: 3600 }` 옵션을 사용하여 1시간 동안 데이터 재검증 없이 캐싱 (ISR)
- API 호출 실패 시 에러를 던지거나 빈 배열을 반환하는 등의 에러 핸들링 로직 포함
- 반환된 데이터의 `results`를 `PokemonListResponse` 타입으로 변환하여 반환
