```markdown
### 파일: src/app/api/pokemon/route.ts
---
설명: 포켓몬 목록 데이터를 제공하는 API 핸들러
- HTTP GET 요청을 처리합니다.
- 쿼리 파라미터(`offset`, `limit`)를 받아 데이터를 페이징합니다.
- PokeAPI(`https://pokeapi.co/api/v2/pokemon`)에서 데이터를 가져옵.
- 응답 데이터 구조: `{ name, id, url, image }` (image URL은 official-artwork 사용)
- CORS 및 에러 핸들링을 포함합니다.

### 파일: src/lib/api/pokemon.ts
---
설명: PokeAPI 호출 및 데이터 가공을 담당하는 유틸리티 모듈
- `fetchPokemonList(offset, limit)` 함수를 export 합니다.
- 외부 API 호출 로직을 분리하여 테스트와 재사용성을 높입니다.
- 받아온 데이터에서 필요한 필드(ID, 이미지 URL)를 추출하고 가공합니다.
- 이미지 URL: `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/other/official-artwork/{id}.png` 형식으로 생성합니다.

### 파일: src/types/pokemon.ts
---
설명: 포켓몬 데이터의 타입 정의 (TypeScript)
- `PokemonListItem` 인터페이스 정의
  - `id`: number
  - `name`: string
  - `image`: string
  - `types`: string[] (선택 사항이나 향후 확장성 고려)
- API 응답 형식에 대한 타입 보장을 제공합니다.

### 파일: README_TESTING.md
---
설명: API 테스트 가이드 문서
- 브라우저 또는 Postman에서 테스트하는 방법을 명시합니다.
- 테스트 URL: `http://localhost:3000/api/pokemon?offset=0&limit=20`
- 예상 응답 JSON 구조 예시를 포함합니다.
- 이미지 URL이 정상적으로 포함되어 있는지 확인하는 체크리스트를 제공합니다.
```
